What's New?

The February 23, 2021 release was primarily focused on improving the user's ability to use mGIS. Functionality was enhanced to be able to move and duplicate tasks. Reporting on Checklists was improved. Risk management was further enhanced. User's can define mGIS start up preferences. Some performance enhancements were included as well.

Improvements

Added the ability to provide the Progress of a Checklist item

Checklist items were updated to take up less space so that more checklist items can be seen. Additionally, checklist items can be expanded to see the details of the item.

Most importantly, checklist items have been updated to provide the user the ability to add the progress that they are making on the items they are working on within a Checklist. To add a progress update, the user will click on the + next to Progress Update.

After selecting the + next to Progress Update the user will see where they can enter their progress update.

After the user saves, the Progress Update will be added to the Progress table.

Enhanced duplicate and move functionality

The ability to Move and Duplicate items has been consolidated to a right-click menu off of the Sheet table.

  • Open will open the details page of the item same as double-clicking on the row.

  • Duplicate will create a duplicate within this same project with - Copy appended to the name of the item.

  • Duplicate To ... will open another drop-down menu where the user can select to where they want to duplicate the item.

  • Move To ... will open another drop-down menu where the user can select to where they want to move the item.

  • Delete will delete the item after confirming that is what the user wants to occur.

Provided ability to have a Risk Approach approved

The Risk Prioritize page was updated to provide the ability to have a Risk Approach approved.

The assigned Risk Approach Approver must be signed in to approve the approach. Upon selecting the Approved checkbox the Date and Time of the approval will be captured.

Created a user preferences page

Different users can have different responsibilities associated with their use of mGIS. So to assist users in focusing mGIS on where they want to start their work, Preferences and Start On have been added to their User Profile.

The user can bypass the Welcome page by selecting the Program they wish always start their work. If the Program is selected, the user can select one of the Project Groups within the Program, and if a Project Group is selected the user can select the Project they wish be opened first.

Additionally, the user can default their initial Navigation to opening on one of the views: Map, Sheet, Kanban, Gantt, or Team.

Lastly, the user can default their Sub Navigation to be one of the tabs: Overview, Tasks, Issues, Risks, or Notes.

Updated Assigned Users on the Sheet table to have the appropriate R or A next to the user's name

So that the users can tell the role of the assigned users from the Sheet table, R and A were added in front of the user who is Responsible and Accountable.

The roles of Consulted (C) and Informed (I) were not included to save on the landscape of the page and not user to many rows. However, in a near future release a check box will be added to this page that will allow the user to show the C and I users as well when needed.

Updated the Last Update column to distinguish the meta data from the Update

So that users can more easily distinguish the last update from the time and date of the last update, the time and date have been distinguished.

To save on landscape in the table, who made the last update can be seen by hovering over the date and time.

Replaced Modal details edit page associated with the Sheet view

Due to the modal details edit page causing so many issues, it was removed and replaced with an inline edit page.

The Save button no longer closes the page, so the user can now save their changes along the way without have to reopen the details page. To close the details page select the Close button.

Issues Resolved

Project not defaulted to the current project

The Project field was added as an option to be able to move an item to somewhere else. The primary problem with this field was that when adding a new item it was defaulted to the current project. Anyway, better functionality to move items has been put into place so this field has been removed. See Enhanced duplicate and move functionality under Improvements above to additional details on the new move functionality.

Rounding issue

Calculations did not have a rounding rule, so only used the numbers before the decimal place. Standard rounding rules, .5 round up, otherwise round down, are now used.

The delete of a Risk, does not delete the Risk

There was a bug so the delete did not complete, but this has been resolved.

Not able to set Risk "On Watch"

There was a bug such that setting On Watch was not captured and that has been resolved. There is another bug that selecting On Watch closes the page that still needs to be resolved.

On Risks, Probability Description needs a * to show it is required

The * has been added to show the field is required.

Add Task on Map did not display the add form

There was a bug that has been resolved.

Search Risks on Sheet page never found results

There was a bug that has been resolved.

Added Related Risks to Issues

The ability to add related risks to issues was missing and was added.
